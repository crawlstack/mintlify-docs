The `runner` object is available globally in every crawler script. It provides the bridge between the browser's DOM and Crawlstack's backend infrastructure.

## Core Methods

### `publishItems(items)`

Sends extracted data to the database and triggers configured webhooks.

```javascript
await runner.publishItems([
  {
    id: "unique-key-123",
    data: { name: "Product", price: "$10" },
    changefreq: "daily"
  }
]);
```

- **Parameters**: `items: RawItem[]` (See [Types Reference](#types-reference))
- **Returns**: `Promise<BackendCallResult<TItem[]>>`

### `addTasks(links)`

Adds new URLs to the crawl queue.

```javascript
await runner.addTasks([
  { 
    href: "https://example.com/page/2", 
    strategy: "NAVIGATE",
    ctx: { category: "electronics" } 
  }
]);
```

- **Parameters**: `links: TaskDef[]` (See [Types Reference](#types-reference))
- **Returns**: `Promise<BackendCallResult<TTask[]>>`

### `getCurrentTask()`

Returns the metadata and custom context for the currently executing task.

```javascript
const task = await runner.getCurrentTask();
console.log(`Current Depth: ${task.depth}`);
console.log(`Custom Context:`, task.taskDef.ctx);
```

- **Returns**: `Promise<TTask>`

## Human-like Interactions

These methods use the Chrome DevTools Protocol to simulate interactions that are indistinguishable from real user actions.

### `humanClick(element, button?)`
Simulates a trusted mouse click at the element's exact coordinates.

```javascript
const btn = document.querySelector("#submit");
await runner.humanClick(btn, "left");
```

### `humanScrollInView(element)`
Smoothly scrolls the page until the element is visible.

```javascript
const footerLink = document.querySelector("a.footer");
await runner.humanScrollInView(footerLink);
```

## Browser Automation Helpers

### `sleep(min, max?)`
Pauses execution. If `max` is provided, it sleeps for a random duration between `min` and `max` milliseconds.

```javascript
await runner.sleep(1000, 3000); // Sleep for 1-3 seconds
```

### `waitFor(selectorOrFn, options?)`
Waits for an element to appear or a function to return true.

```javascript
await runner.waitFor(".success-message", { timeoutMs: 15000 });
```

### `getByTextDeep(text, options?)`
Finds an element by text content, even if it's nested inside **Shadow DOM**.

```javascript
const button = runner.getByTextDeep("Load More", { exact: true });
button?.click();
```

## Network Interception

### `getRequests()`
Returns all network requests captured during the current page load.

```javascript
const requests = await runner.getRequests();
const apiCall = requests.find(r => r.url.includes("/api/v1/data"));

if (apiCall) {
  const body = await apiCall.body(); // Get response body
  console.log(JSON.parse(body));
}
```

### `clearRequests()`
Clears the current captured requests buffer.

## File Downloads

### `enableDownloads()`
Enables the interception of both network downloads and client-side (Blob) downloads. Captured files are saved to your configured S3 bucket or local OPFS storage.

```javascript
await runner.enableDownloads();
```

### `getDownloads()`
Returns an array of files that have been intercepted during the current task.

```javascript
const downloads = await runner.getDownloads();
for (const file of downloads.files) {
    console.log(`URL: ${file.url}, Status: ${file.status}`);
}
```

- **Returns**: `Promise<{ files: { url: string, status: 'pending'|'done'|'failed', error?: string }[] }>`

### `clearDownloads()`
Clears the list of tracked downloads from the current task's queue.

```javascript
await runner.clearDownloads();
```

## Real-time Events

### `captureWebsockets(callback)`
Registers a listener for all WebSocket traffic in the current tab. The callback is called for every frame sent or received.

```javascript
await runner.captureWebsockets(async (event) => {
    console.log(`Socket [${event.type}]: ${event.message}`);
});
```

- **Parameters**: `callback: (event: WebSocketEvent) => void | Promise<void>`
- **Returns**: `Promise<void>`

### `captureSse(callback)`
Registers a listener for Server-Sent Events (SSE). The callback is called for every event received on an `EventSource` connection.

```javascript
await runner.captureSse(async (event) => {
    console.log(`SSE Event: ${event.message}`);
});
```

- **Parameters**: `callback: (event: SseEvent) => void | Promise<void>`
- **Returns**: `Promise<void>`

## Types Reference

### `RawItem`
The format for data you want to save.

| Field | Type | Description |
| :--- | :--- | :--- |
| `id` | `string` | **Required.** Unique key for deduplication. |
| `data` | `object` | **Required.** The extraction payload (JSON). |
| `changefreq` | `string` | Optional. Hints for re-crawling (`always`, `daily`, `never`, etc). |

### `TaskDef`
The definition for a new URL to crawl.

| Field | Type | Description |
| :--- | :--- | :--- |
| `href` | `string` | **Required.** The destination URL. |
| `strategy` | `string` | `NAVIGATE` (default) or `REOPEN` (fresh browser context). |
| `ctx` | `any` | Custom data to pass to the script running on the child page. |
| `changefreq` | `string` | Frequency for re-visiting this link. |

### `TTask`
The task object returned by `getCurrentTask()`.

| Field | Type | Description |
| :--- | :--- | :--- |
| `url` | `string` | The URL being crawled. |
| `depth` | `number` | The current crawl depth. |
| `status` | `string` | `queued`, `processing`, `success`, `failed`, etc. |
| `taskDef` | `TaskDef` | The original definition used to create this task. |

### `NetworkRequest`
Captured network metadata.

| Field | Type | Description |
| :--- | :--- | :--- |
| `url` | `string` | Request URL. |
| `method` | `string` | HTTP method (GET, POST, etc). |
| `status` | `number` | HTTP response status code. |
| `body()` | `function` | Async function that returns the response body as a string. |

### `WebSocketEvent`
Captured WebSocket frame metadata.

| Field | Type | Description |
| :--- | :--- | :--- |
| `url` | `string` | The URL of the WebSocket connection. |
| `type` | `string` | `send` or `receive`. |
| `message` | `string` | The payload data of the frame. |
| `opcode` | `number` | The WebSocket opcode (1=text, 2=binary). |
| `timestamp` | `number` | Browser timestamp of the event. |

### `SseEvent`
Captured Server-Sent Event metadata.

| Field | Type | Description |
| :--- | :--- | :--- |
| `url` | `string` | The URL of the event stream. |
| `eventName` | `string` | The name of the event. |
| `eventId` | `string` | The ID of the event. |
| `message` | `string` | The payload data. |
| `timestamp` | `number` | Browser timestamp of the event. |
